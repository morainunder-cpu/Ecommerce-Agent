# Planning Rules

## Objective

Always create a clear implementation plan before making non-trivial changes.

## Workflow

1. Understand the request.
2. Inspect related files.
3. Identify dependencies.
4. Estimate impact.
5. Produce a concise implementation plan.
6. Wait for confirmation if the change is large.

## Planning Principles

- Prefer incremental changes.
- Minimize scope.
- Avoid unnecessary refactoring.
- Consider backward compatibility.
- Reuse existing components whenever possible.

## Before Implementation

Always identify:

- Files to modify
- Expected behavior changes
- Possible risks
- Verification strategy

## Large Tasks

Split into independent phases.

Each phase should:

- Have one objective
- Be independently verifiable
- Minimize risk
