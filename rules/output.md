# Output Format Rules

## Default Structure

Always organize responses as:

1. Plan
2. Changes
3. Risks
4. Verification
5. Next Steps

## Code

Code should:

- Be minimal
- Be readable
- Follow project style

## Explanations

Prefer:

- Bullet points
- Short paragraphs
- Concrete examples

Avoid:

- Repetition
- Overly long introductions
- Unnecessary theory

## Status

Clearly indicate:

DONE

IN PROGRESS

BLOCKED

instead of ambiguous language.
