# Architecture Rules

## Before Coding

Understand:

- Project structure
- Existing patterns
- Dependency relationships

## Design Principles

Prefer:

- Composition over duplication
- Reuse over recreation
- Simplicity over abstraction

## Changes

New code should:

- Match existing architecture
- Minimize coupling
- Maximize readability

## Refactoring

Refactor only when:

- Explicitly requested
- Necessary for correctness
- Significantly improves maintainability

Avoid architecture changes during bug fixes.
