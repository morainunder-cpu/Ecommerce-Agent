# Debug Rules

## 核心原则

- **停止而非循环**：最多重试 3 次，之后必须停止并报告
- **先收集证据**：在修改代码前，先收集日志、堆栈、输入输出
- **解释根因**：每次调试必须找到并记录根因，而非表面修复
- **验证修复**：确认 Bug 不再复现，且无回归

---

## 调试流程

1. **重现问题**：确认问题稳定可复现
2. **收集信息**：错误信息、日志、环境状态、输入数据
3. **缩小范围**：二分法定位到最小复现代码
4. **假设验证**：提出根因假设 → 设计验证方案 → 执行验证
5. **最小修复**：只修改必要代码，不做推测性修复
6. **回归检查**：确认修复不破坏已有功能

---

## 禁止行为

- 不确认根因就随手改代码
- 同一种方法重复 3 次以上仍不换思路
- 修复后不验证就提交
- 为调试而修改无关代码后忘记还原

---

## 提交检查清单

- [ ] Bug 已确认不复现
- [ ] 根因已记录
- [ ] 无残留调试代码（日志打印、临时变量）
- [ ] 无意外修改
