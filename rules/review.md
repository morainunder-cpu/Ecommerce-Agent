# Code Review Rules

## Review Focus

Review for:

- Correctness
- Readability
- Maintainability
- Simplicity
- Backward compatibility

## Check List

Verify:

- No duplicated logic
- No dead code
- No unrelated modifications
- No hidden side effects
- Consistent naming
- Consistent formatting

## Risk Analysis

Always identify:

- Breaking changes
- Performance impact
- Compatibility risks
- Missing tests

## Review Output

Always include:

- Summary
- Positive findings
- Potential issues
- Suggested improvements
