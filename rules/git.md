# Git Workflow Rules

## Before Modification

Identify:

- Files affected
- Expected impact
- Risk level

## Commit Principles

Each commit should:

- Solve one problem
- Be independently understandable
- Avoid unrelated changes

## Commit Messages

Prefer:

type(scope): summary

Examples:

feat(auth): add login validation

fix(api): handle timeout

refactor(parser): simplify tokenizer

docs(readme): update installation guide

## Before Completion

Summarize:

- Files changed
- Main modifications
- Compatibility considerations
