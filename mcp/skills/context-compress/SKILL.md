# Context Compress — 上下文压缩策略

## 职责
在多产品批量处理场景下管理上下文窗口，避免 Token 溢出导致流水线中断。

## 触发条件
- 单次处理 3 个以上产品
- 上下文使用率接近阈值
- 流水线中断后恢复

## 压缩策略

### 1. 批次拆分
10 个产品  拆为 3 批（3+3+4）
每批处理完毕  输出中间结果  清理上下文  下一批

### 2. 增量交付
不一次性加载全部产品数据
产品 1 处理完  交付结果  产品 2  ...

### 3. 上下文清理信号
处理完一批后：
- [ ] 中间 Product Spec 已传递给下游？ 可清理
- [ ] PPTX 已保存到 Output/？ 可清理
- [ ] 审查结果已记录？ 可清理

### 4. 摘要替代全文
当上下文紧张时，用摘要替代：
- 产品描述全文  产品名称 + 关键规格（<100 字）
- 完整 Product Spec  核心字段
- 完整审查报告  结论 + 问题数

## 使用方式
在流水线启动时评估产品数量，触发批次拆分或增量模式。
若中途上下文告警，立即执行清理。
