# 完整工作流示例

## 场景

新项目接入 → 功能开发 → 代码审查 → 发布。

## 步骤

### 1. 项目入职

参照 `playbooks/project_onboarding.md`：
- 了解目录结构
- 识别技术栈
- 发现编码规范

### 2. 功能开发

参照 `playbooks/full_feature.md`：
- Phase 1: 需求分析
- Phase 2: 项目审查
- Phase 3: 规划
- Phase 4: 风险评估
- Phase 5: 实施
- Phase 6: 自我审查
- Phase 7: 验证
- Phase 8: 最终报告

### 3. 代码审查

参照 `playbooks/code_review.md`：
- 检查正确性、可读性、可维护性
- 严重度分类（Critical/High/Medium/Low）

### 4. 发布

参照 `playbooks/release_process.md`：
- 版本号更新
- Changelog 更新
- 回滚方案准备
