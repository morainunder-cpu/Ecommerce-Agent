# Glossary

## AI Agent 术语

### Agent
核心 AI 实体，能够通过交互工具执行软件开发任务，包括文件操作、代码编辑、浏览器操作等。

### Agent State
Agent 的当前上下文和状态，包括其记忆（memory）、活动工具和正在进行的任务。

### Prompt
向语言模型发送的指令文本，用于引导模型生成特定输出。包含 system prompt、user prompt 等类型。

### System Prompt
定义 AI 行为边界和角色的顶层指令，通常在对话开始时设置。

### Context Window
语言模型一次可处理的 Token 数量上限，超出部分需要截断或压缩。

### Chain of Thought (CoT)
引导模型逐步推理的提示技术，提高复杂问题的回答质量。

### Tool Calling
语言模型调用外部工具的能力，如执行命令、读写文件、访问网络等。

### RAG (Retrieval-Augmented Generation)
检索增强生成，通过检索外部知识库来增强模型回答的准确性。

### Fine-tuning
在预训练模型基础上使用特定领域数据继续训练，使其适应特定任务。

## 电商术语

### SKU (Stock Keeping Unit)
库存量单位，用于唯一标识商品的编码系统。

### SPU (Standard Product Unit)
标准化产品单元，代表一类商品的集合，如同款不同色的夹克。

### Gem Image
京东平台提供的 AI 商品图生成工具，基于用户输入的 Prompt 生成商品展示图。

### Prompt Engineering
设计、优化输入提示词的技术，使 AI 模型生成更符合需求的输出。

### 四宫格细节图
将商品展示图分割为四个区域，分别展示不同细节部位（如反光条、拉链、袖口、口袋）。

### 电商摄影
专门用于商品展示的摄影风格，强调产品清晰度、材质真实感和商业可用性。

### 自然光
模拟自然光照条件的摄影布光方式，使产品图更真实、柔和。

### PPTX 目录
使用 PowerPoint 格式制作的商品目录，用于电商展示或客户预览。

## 项目结构术语

### AGENTS.md
项目根级别的 AI 行为指令文件，定义 AI 在项目中的工作方式和约束。

### Rules
目录 `rules/`，包含 AI 在项目中的行为规则文件，覆盖编码、安全、规划、审查等方面。

### Playbooks
目录 `playbooks/`，包含标准操作流程文档，如代码审查流程、紧急 Bug 修复流程、功能开发流程等。

### Templates
目录 `templates/`，包含各类文档的标准模板，用于规范输出格式。

### Prompts
目录 `prompts/`，包含常用提示词模板，用于快速启动特定类型的工作。

### Knowledge
目录 `knowledge/`，包含项目知识库文件，如架构说明、编码约定、术语表等。

---

## 流程术语

### Onboarding
新成员（或 AI）加入项目时的引导流程，包括环境搭建、项目理解、规范学习。

### Code Review
代码审查流程，系统性地检查代码的正确性、可维护性、安全性等。

### Incident Response
事故响应流程，对生产环境问题或严重 Bug 进行快速处理和恢复。

### Release Process
发布流程，包含版本号规范、变更日志、构建、测试、部署等步骤。

### Rollback
回滚操作，将系统或代码恢复到之前的稳定版本。
